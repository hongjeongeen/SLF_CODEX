# Customer - 보험길잡이 AGENT

## ICP
- Primary: 삼성생명 다이렉트에서 `암/건강보험`을 실제로 비교하며 가입하려는 40~50대 실수요자
- Secondary: 가입 흐름 개선, 이탈 감소, 설명 품질 향상을 고민하는 보험 디지털 기획 / UX / CX 담당자

## Persona
| | A. 40~50대 실수요자 | B. 디지털 보험 기획자 | C. 상담 / CX 운영 담당 |
|---|---|---|---|
| 핵심 목표 | 지금 보는 특약이 왜 필요한지 빠르게 이해하고 가입을 이어가기 | 어려운 툴팁을 맥락형 설명 경험으로 바꾸기 | 설명으로 해결되지 않는 고객만 효율적으로 상담 연결 |
| 현재 문제 | 용어가 어렵고, 툴팁이 길고, 차이가 헷갈린다 | 기존 툴팁은 약관 요약이라 UX 효과가 낮다 | 고객이 무엇을 보고 막혔는지 맥락 없이 상담이 들어온다 |
| 현재 행동 | 읽다 포기, 싼 안만 선택, 나중에 알아보기, 상담으로 이동 | 텍스트 개선과 화면 개편 사이에서 우선순위 고민 | 반복 설명, 고객 의도 재확인, 상담 리드타임 증가 |
| 원하는 결과 | `아 이건 이런 의미구나`를 바로 이해 | 설계 지속률과 이해도를 함께 올리기 | 현재 항목과 막힌 포인트가 붙은 상담 연결 |

## Primary Persona Detail
- 나이대: 40~50대
- 상황: 본인 또는 배우자 보장을 점검하며 모바일에서 다이렉트 보험을 비교
- 태도: 보험 필요성은 인정하지만 특약 구조와 용어는 어렵게 느낌
- 불안: 잘못 선택할까 걱정, 괜히 보험료만 올라갈까 걱정, 상담 연결 시 권유가 부담될까 걱정
- 기대: 쉬운 설명, 비슷한 특약 간 차이, 부담 없는 상담 확인

## Top Pain Points
1. `통합암`, `소액암`, `비급여` 같은 용어가 낯설다
2. 비슷한 특약이 많아 무엇이 다른지 모르겠다
3. 금액을 올리거나 특약을 넣었을 때 무엇이 달라지는지 감이 안 온다
4. 현재 툴팁은 길고 읽기 부담스럽다
5. 상담이 필요해도 처음부터 다시 설명해야 할까 봐 귀찮다

## Desired Moments
- 용어를 눌렀더니 바로 쉬운 말로 바뀐다
- 토글을 켰더니 왜 이 선택이 의미 있는지 짧게 알려준다
- `일반암과 차이`를 표처럼 보여줘서 빨리 이해된다
- `국민건강보험이 적용되지 않는 치료인지`처럼 실제 병원비 기준으로 풀어줘서 덜 헷갈린다
- `상급종합병원`이 정확히 어떤 병원인지 자세히 보기에서 확인할 수 있다
- 필요하면 `이 항목만 상담받기`로 이어질 수 있다

## Buying / Adoption Signals
- 고객이 설명 열람 후 다음 단계로 바로 이동한다
- 비교 보기 클릭 후 토글 상태를 유지하거나 조정한다
- 상담 연결 시 현재 항목 맥락이 함께 넘어가 상담이 짧아진다
- 내부 기획자가 새 용어/특약을 데이터로 추가해 빠르게 확장할 수 있다

## Research Questions
1. 실제로 고객이 가장 많이 멈추는 용어와 특약은 무엇인가
2. `쉬운 설명`과 `비교 설명` 중 어느 쪽이 선택 지속에 더 영향을 주는가
3. 자동 설명 카드의 적정 노출 빈도는 어느 정도인가
4. 상담 연결 문구 중 고객 부담을 가장 낮추는 표현은 무엇인가
