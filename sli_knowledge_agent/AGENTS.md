# AGENTS.md - 보험길잡이 AGENT 작업 지침

## Context
삼성생명 다이렉트 보험 가입 과정에서 고객이 어려워하는 보험 용어와 특약 구조를 쉬운 말로 설명하는 `보험길잡이 AGENT` 프로젝트다. 목적은 단순 용어사전이 아니라, 가입 도중 막히는 순간에 현재 보고 있는 보장 맥락에 맞는 설명을 제공해 설계를 계속 이어가게 돕는 것이다.

## Goal
고객이 `통합암`, `소액암`, `비급여`, `상급종합병원`, `직접치료` 같은 용어에서 멈췄을 때 다음 3가지를 빠르게 제공한다.

1. 쉬운 한 줄 설명
2. 비슷한 보장과의 차이
3. 필요 시 부담 없는 상담 연결

## Product Principles
- 툴팁보다 이해를 우선한다.
- 정의보다 의사결정에 필요한 설명을 먼저 준다.
- 쉬운 말로 설명하되, 약관 의미를 왜곡하지 않는다.
- 추천처럼 보이기보다 판단 기준을 제공한다.
- 상담 연결은 실패가 아니라 자연스러운 다음 단계로 본다.

## MVP Scope
- 가입 흐름 내 `암/건강보험` 맥락 설명
- 용어 클릭 시 쉬운 설명
- 옵션 ON/OFF, 가입금액 변경 시 맥락형 설명 카드
- 비슷한 특약 간 비교 보기
- `권유 없이 필요한 내용만 상담해드려요 (1588-0000)` 연결

## Out of Scope
- 개인 건강정보 기반 맞춤 추천
- 상품 추천, 플랜 추천, 최적 설계 자동화
- 언더라이팅 판단, 가입 가능 여부 판단
- 약관을 대체하는 법적 해석

## WHETHER before HOW
이 Agent가 답해야 할 질문은 항상 아래 순서를 따른다.

1. 고객이 지금 무엇을 보고 있는가
2. 고객이 무엇이 헷갈려서 멈췄는가
3. 이 순간 설명이 필요한가, 비교가 필요한가, 상담 연결이 필요한가

기술 구현보다 먼저 현재 화면 맥락과 고객 질문 단위를 맞추는 것이 우선이다.

## UX Rules
- 플로팅 아이콘은 사용하지 않는다.
- 하이라이트는 `특약명`과 `핵심 어려운 용어`에만 제한한다.
- 하이라이트 애니메이션은 첫 노출 시 1회만 짧게 사용한다.
- 자동 설명 카드는 같은 항목에 대해 과도하게 반복 노출하지 않는다.
- 기본 패턴은 `sticky guide bar + bottom sheet` 혼합형을 우선 검토한다.

## Content Rules
- 기본 설명 구조는 `한 줄 요약 -> 왜 중요한가 -> 이런 분께 맞음 -> 주의할 점`이다.
- 설명 문장은 2~3문장 단위로 짧게 끊는다.
- `꼭 필요하다`, `유리하다`, `추천한다` 같은 권유형 표현은 피한다.
- `검토할 수 있다`, `이렇게 비교해보면 된다` 같은 판단 보조형 표현을 사용한다.
- 모든 상세 설명에는 `자세한 기준은 약관을 꼭 확인해 주세요` 성격의 문구를 포함한다.
- 약관 기준이 중요한 항목은 `세부 기준은 약관을 따른다`를 명시한다.
- `비급여`는 `국민건강보험이 적용되지 않는 치료`로 풀어쓴다.
- `상급종합병원`은 `큰 병원`처럼 뭉뚱그려 단정하지 않고, 자세히 보기에서 상급종합병원으로 분류된 병원 예시와 함께 `자세한 기준은 약관을 꼭 확인해 주세요`를 함께 안내한다.

## Data Rules
콘텐츠는 최소 2계층으로 관리한다.

1. `term dictionary`
용어별 기본 쉬운 설명, 주의점, 비교 대상

2. `event guidance`
토글 ON/OFF, 가입금액 변경, 비교 진입 같은 이벤트별 카드 문구와 노출 룰

같은 용어라도 상품 맥락에 따라 설명 포인트가 달라질 수 있으므로, 완전한 1:1 사전만으로 끝내지 않는다.

## HITL / Failure
- 설명만으로 판단이 어렵거나 민감한 질문은 상담으로 연결한다.
- 보장 범위, 지급 조건, 예외가 약관 의존적인 경우 단정하지 않는다.
- 근거 데이터가 없거나 맥락 인식이 불확실하면 짐작해서 답하지 않는다.
- 고객이 원하면 `지금 보고 있는 항목만` 상담으로 넘긴다.

## Success Check
- 용어 설명 열람 후 다음 단계 진행률이 개선되는가
- 설명 후 옵션 선택 지속률이 개선되는가
- 같은 화면에서 이탈률이 줄어드는가
- 상담 연결이 맥락형으로 자연스럽게 이어지는가
